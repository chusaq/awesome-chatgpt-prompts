# CLAUDE.md — Codebase Guide for AI Assistants

## Repository Overview

**awesome-chatgpt-prompts** is a community-curated collection of role-playing and persona prompts for ChatGPT (and similar LLMs). The repository has no application code — it is a content repository with two canonical data files and a static Jekyll site published via GitHub Pages.

- GitHub: `f/awesome-chatgpt-prompts`
- Live site: `prompts.chat` (CNAME)
- Hugging Face mirror: `fka/awesome-chatgpt-prompts`

## The Two Canonical Data Files

Every prompt **must appear in both files**. They are kept in sync manually.

### 1. `README.md` — Human-readable prompts

Each prompt entry follows this exact structure:

```markdown
## Act as a [Role Title]
Contributed by: [@username](https://github.com/username)

> Prompt text goes here.
```

Optional lines between the heading and the blockquote:

```markdown
**Alternative to**: Grammarly, Google Translate
**Examples**: Node.js Backend, React Frontend Developer
<mark>Generated by ChatGPT</mark>
```

Rules:
- The heading **must** start with `## Act as` (two `#` marks).
- The prompt text **must** be in a `>` blockquote.
- The `Contributed by` line must include a GitHub profile link.
- If the prompt was generated by ChatGPT, append `<mark>Generated by ChatGPT</mark>` to the contribution line.

### 2. `prompts.csv` — Machine-readable prompts

**Header (line 1):** `"act","prompt"`

**Schema:**

| Column | Description |
|--------|-------------|
| `act`  | The role title without the "Act as" prefix, double-quoted |
| `prompt` | The full prompt text, double-quoted |

**Encoding rules:**
- All values must be wrapped in double-quotes.
- Internal double-quotes are escaped by doubling: `""like this""`.
- No trailing whitespace on any line (CI enforces this).
- UTF-8 encoding, quoting style 2 (as expected by csvkit).

**Example row:**
```csv
"Linux Terminal","I want you to act as a linux terminal. ..."
"English Translator and Improver","... My first sentence is ""istanbulu cok seviyom"""
```

## Adding a New Prompt

1. **Edit `README.md`** — append the new entry at the end of the Prompts section using the heading/blockquote format above.

2. **Edit `prompts.csv`** — append a new row:
   - Strip "Act as a/an " from the title for the `act` column.
   - Wrap both values in double-quotes.
   - Escape any double-quotes inside the text by doubling them.
   - Do **not** add trailing whitespace.

3. **Verify locally** (optional but recommended):
   ```bash
   pip install csvkit
   csvclean -n -u 2 prompts.csv
   grep -P "[[:space:]]$" prompts.csv  # should return nothing
   ```

## CI — `csv_linter.yml`

Runs on every `push` and `pull_request`. Steps:

1. **Install csvkit** (`pip install csvkit`)
2. **Lint CSV** — `csvclean -n -u 2 prompts.csv` (dry-run, UTF-8 quoting)
3. **Fail on errors** — exits non-zero if csvclean reports any problems
4. **Check trailing whitespace** — `grep "[[:space:]]$" prompts.csv` must produce no output

A PR cannot be merged if CI fails. Fix the CSV before pushing.

## Jekyll / GitHub Pages

The site is built automatically by GitHub Pages from the `main` branch.

- `_config.yml` sets the site name; all other Jekyll settings use GitHub Pages defaults.
- `_layouts/default.html` adds:
  - **Copy-to-clipboard** button injected before each `<blockquote>` under an `h2[id^=act]` heading.
  - **Google Analytics** (tag `G-MSNHFWTE77`).
  - **anchor-js** for heading permalinks.
- No build step is needed locally; GitHub Pages builds on push.

## Contribution Conventions

- **PR checklist** (`.github/pull_request_template.md`) must be completed:
  - [ ] Prompt tested and working
  - [ ] `Contributed by` line with GitHub username added to README.md
  - [ ] Entry added to README.md
  - [ ] Entry added to prompts.csv
  - [ ] Quotes in CSV escaped by doubling
  - [ ] "Act as" removed from CSV `act` column title

- **No code changes** are expected in normal contributions — this is a content-only repo.
- **Quote escaping** is the most common source of CI failures; always double internal quotes in the CSV.
- **No trailing whitespace** — editors that auto-strip trailing whitespace are recommended.

## Key Invariants

- `README.md` and `prompts.csv` must always be in sync (same set of prompts).
- The `act` column in CSV is always the title without the "Act as a/an" prefix.
- CSV must pass `csvclean -n -u 2` with zero errors.
- No trailing whitespace anywhere in `prompts.csv`.
- Every blockquote in README.md corresponds to exactly one CSV row.
